Задание 09 (Использвоание семафора SYSTEM V)

Скорректировать программу 08 так, чтобы несколько процессов (ограниченное количество) могли читать из файла. 
Запись в файл возможна, когда он никем не читается.


Использовани семафоров SYSTEM V
При заупуске необходимо ввести аргумент - количесвто случайных чисел, создаваемых дочерним процессом.
Логика работы:
Порождается первый дочерний процесс, который генерирует случайное число и записывает в канал.
Затем порождаются CH_PR проессов родительским процессом, эти процессы будут читать данные из файла

Создается три семафора: 
- нулевой (0) - отвечает кол-во процессов которые читают файл в данный момент времени   (максимальное кол-во одновременно читающиъх процессов = NUM_ACCESS)
- первый (1) - доступ к чтению (0/1)
- второй (2) - кол-во процессов, которым осталось прочитать число (когда прочтут все значение станет равным 1)

Действия с семаформаи описаны в программе, для каждого процесса

Родительский процесс читает даынные из канала, выводит их на экран и записывает в файл "numbers" в двоичном формате, 
затем открывает доступ(1м семафором) на чтение, ждет пока все процессы не прочтут (никто не будет читать файл) (значение 2го семафора станет равным 1),
закрывает доступ (1м семафором), переводит 2ий семафор в нулевое значение, давая дочерним процессам перейти на следующую итерацию.

В это время дочерний процесс, находился в режиме ожидания, пока значение 1го семафора не станет равным 0.
Дочерний процесс, проверяя возможность подключения к чтению(кол-во процессов которые могут одновременно читать файл ограниченно нулевым (0) семафором) 
считывает данные из файла и выводит их на экран, освобождает "возможность чтения", уменьшает значение 2го семафора на -1 (отметка о прочитке)
Так повторятся n-ое количесвто раз.
